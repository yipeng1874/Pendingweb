# 最终上线配置索引文档

## 1. 文档目标

本文档用于汇总当前 `D:\Pendingweb` 项目在“公司电脑 + FRP + Nginx + 独立 PC/H5 入口”方案下的最终上线关键配置。

目标是让后续维护时，不必在多个文件之间来回查找，而是通过本文档快速定位：

- 当前正式访问入口是什么
- FRP 映射到哪里
- Nginx 监听哪些端口
- 后端环境变量要配什么
- PC / H5 飞书回调地址分别是什么
- 相关配置文件分别在哪里

---

## 2. 当前推荐正式访问架构

推荐正式访问链路如下：

```text
PC 用户浏览器
  -> FRP 外网端口 29266
  -> 本机 / 公司电脑 Nginx 8088
  -> frontend/dist
  -> /api 代理到 backend 4000

H5 用户浏览器
  -> FRP 外网端口 29267
  -> 本机 / 公司电脑 Nginx 8081
  -> todo-h5/dist
  -> /api 代理到 backend 4000
```

说明：

- PC 与 H5 使用独立入口
- 不再让 H5 挂在 `/h5/` 子路径下
- 这样更稳定，也更容易维护飞书回调链路

正式访问模式需要的端口：

- H5 Nginx：`8081`

正式访问时不需要：

- `5173`
- `4173`

---

## 6. FRP 当前推荐映射

配置文件位置：

- `D:\Pendingweb\frpc.ini`

当前推荐保留：

### 6.1 开发映射（可保留用于调试）

- `29263 -> 5173`（PC Vite）
- `29264 -> 4000`（Backend）
- `29265 -> 4173`（H5 Vite）

### 6.2 正式访问映射（推荐）

- `29266 -> 8088`（PC Nginx）
- `29267 -> 8081`（H5 Nginx）

建议：

- 开发阶段可同时保留开发映射和正式映射
- 后续正式使用优先走 `29266` / `29267`

---

## 7. Nginx 当前关键职责

Nginx 配置文件位置：

- `C:\nginx-1.30.2\conf\nginx.conf`

当前职责：

### 7.1 PC 入口（8088）

- 提供 `frontend/dist`
- 处理 PC 前端路由刷新
- `/api` 代理到 `127.0.0.1:4000`
- `/uploads` 映射上传目录
- PWA sw.js / manifest.webmanifest 不缓存

### 7.2 H5 入口（8081）

- 提供 `todo-h5/dist`
- 处理 H5 路由刷新
- `/api` 代理到 `127.0.0.1:4000`
- `/uploads` 映射上传目录

### 7.3 通用能力

- gzip 压缩
- 静态资源缓存
- HTML 不长缓存
- PWA Service Worker 不缓存（`sw.js`、`manifest.webmanifest`）

## 10. 当前飞书回调配置

### 10.1 PC 飞书回调

建议配置为：

- `FEISHU_REDIRECT_URI_PC=http://frp7.ccszxc.site:29266/pc/auth/callback`

作用：

- PC 飞书网页登录完成后回到 PC 回调页

### 10.2 H5 飞书回调

建议配置为：

- `FEISHU_REDIRECT_URI_H5=http://frp7.ccszxc.site:29267/auth/callback`

作用：

- H5 飞书网页登录完成后回到 H5 自己的回调页

### 10.3 兜底回调

目前仍保留：

- `FEISHU_REDIRECT_URI`

说明：

- 它作为兜底保留
- 推荐正式环境优先使用 PC/H5 分离回调地址

---

## 11. H5 飞书登录链路关键点

当前 H5 已补齐独立飞书登录闭环：

### 11.1 H5 登录发起

H5 会发起：

- `/api/auth/feishu/login?action=login&client=h5&configId=...`

### 11.2 后端识别来源端

后端根据：

- `client=h5`

选择 H5 回调地址。

### 11.3 H5 自己的回调页

H5 当前已新增回调页：

- 路由：`/auth/callback`

外网完整地址：

- `http://frp7.ccszxc.site:29267/auth/callback`

作用：

- 接收飞书 `code` / `state`
- 调用后端完成登录
- 写入 H5 登录态
- 跳转到 `/identity` 或 `/todos`

## 13. 更新项目后的最小操作顺序

如果只是正常更新代码，建议顺序如下：

### 13.1 更新源码

```bash
git pull
```

### 13.2 重新 build

#### PC

```bash
npm run build -w frontend
```

#### H5

```bash
cd todo-h5
npm run build
```

#### Backend（如果后端有改动）

```bash
npm run build -w backend
```

### 13.3 重启 / reload

#### 后端改了

- 重启后端 / PM2

#### Nginx 配置改了

```bash
nginx -s reload
```

#### FRP 配置改了

- 重启 frpc

---

## 14. 哪些情况必须重启

### 14.1 后端代码或 `.env` 改了

必须：

- 重启后端

### 14.2 H5 / PC 页面改了

必须：

- 重新 build 前端

### 14.3 Nginx 配置改了

必须：

- `nginx -s reload`

### 14.4 `frpc.ini` 改了

必须：

- 重启 frpc

## 15. 当前最终关键结论

当前项目经过本机验证后，最终推荐采用：

1. PC 使用独立端口 `8088`（本机 8080 被其他服务占用）
2. H5 使用独立端口 `8081`
3. FRP 映射：
   - `29266 -> 8088`
   - `29267 -> 8081`
4. 后端统一使用 `4000`
5. H5 不再使用 `/h5/` 子路径部署
6. H5 飞书登录使用独立回调页：
   - `/auth/callback`
7. 正式访问时不再需要 `5173` / `4173`
